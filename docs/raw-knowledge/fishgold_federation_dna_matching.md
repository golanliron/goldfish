# Goldfish — Federation DNA Matching (Creative Angles)

> **Updated:** 2026-05-06 | **Purpose:** Smart matching between ANY nonprofit and Jewish federations
> **Rule:** Never reject a match before finding a creative angle (Behavior Rule #11)

---

## How It Works

Each federation has:
1. **Primary DNA** — the obvious categories (education, youth, Israel)
2. **Creative Angles** — secondary connections that open doors for MORE orgs
3. **Funded Orgs Examples** — real organizations they funded (proves the angle works)
4. **Match Triggers** — keywords in an org's profile that should trigger this federation

The matching algorithm should:
1. Check primary DNA match (score 30-100)
2. Check creative angles (score 15-60)
3. Suggest the ANGLE to the org: "Federation X funds Ethiopian programs. Your work with immigrant youth connects through the integration angle."

## Matching Algorithm — How Goldfish Should Use This

```
function matchOrgToFederation(org, federation):

  // Step 1: Primary match (0-40 points)
  primaryScore = matchDNA(org.populations, fed.primaryPopulations) * 20
                + matchDNA(org.domains, fed.primaryDomains) * 20

  // Step 2: Creative angle match (0-40 points)
  for each angle in fed.creativeAngles:
    if org.profile contains angle.triggerKeywords:
      creativeScore += 20
      suggestedAngle = angle.why  // "Your work with Ethiopian youth connects..."

  // Step 3: Israel eligibility bonus (0-20 points)
  if fed.fundsIsrael AND org.country == 'Israel':
    israelBonus = 20

  // Total: 0-100
  return { score, suggestedAngle, contactEmail }
```

## Key Insight for Goldfish Product

The creative angle matching is what makes Goldfish VALUABLE. Any nonprofit can google "Jewish federation grants." But only Goldfish can say:

> "Phoenix CJP funded Ethiopian National Project last year. Your organization works with immigrant youth in Be'er Sheva. There's a connection — they care about integration through education. Here's the contact: info@phoenixcjp.org. Want me to draft the pitch?"

This is the difference between a database and an intelligent agent.
