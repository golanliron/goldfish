# Goldfish — מערכת התראות מסמכים
> עודכן: 2026-05-06

## מה נבנה
מערכת שמזהה מסמכים חסרים, שפג תוקפם, או עומדים לפוג — ומתריעה לעמותות באופן יזום.

## רכיבים

### 1. OrgTab.tsx — UI התראות (שורות 273-321, 564+)
- `DocAlert` interface: type (expired/expiring/missing), label, detail
- בדיקת `metadata.valid_until` בכל מסמך → expired (פג) / expiring (<90 ימים)
- בדיקת 8 `REQUIRED_DOCS` patterns מול מסמכי הארגון → missing
- UI: קופסה אדומה עם נקודות צבע (אדום/כתום/אפור)

### 2. Chat route.ts — AI awareness (loadAllChunks, שורה ~435+)
- שאילתת documents כוללת עכשיו `metadata`
- אותה לוגיקת alerts רצה בצד שרת
- מוסיף לסיסטם פרומפט סקציית "התראות מסמכים" עם הנחיות יזומיות
- Tab focus "העמותה" כולל הנחיות על דרישות מסמכים

### 3. Supabase documents table
- metadata.valid_until: תאריך תפוגה (ISO string)
- metadata.doc_type: סוג מסמך (עברית)
- metadata.summary: תיאור קצר

## 8 מסמכים נדרשים (REQUIRED_DOCS)
1. אישור ניהול תקין (רשם העמותות) — pattern: `/ניהול תקין/`
2. אישור סעיף 46 (פטור ממס לתורמים) — pattern: `/סעיף 46|saif.?46/`
3. אישור ניכוי מס במקור — pattern: `/ניכוי מס/`
4. תעודת רישום עמותה — pattern: `/רישום עמותה|תעודת רישום/`
5. דוח כספי מבוקר — pattern: `/דוח כספי|כספי.*מבוקר/`
6. תקציב מאושר — pattern: `/תקציב.*מאושר|מאושר.*תקציב/`
7. דוח מילולי / דוח פעילות — pattern: `/מילולי|דוח פעילות/`
8. אישור ניהול ספרים — pattern: `/ניהול ספרים/`
