## הרעיון

Goldfish צריך לדעת לצטט **נתונים אמיתיים ומעודכנים** (2024-2026) על כל בעיה חברתית בישראל.
כשעמותה כותבת הגשה — Goldfish מספק רקע מחקרי מבוסס עם מקורות.

**כלל ברזל מוחלט: אפס המצאות. רק מקורות מאומתים.**

## איך Goldfish משתמש בזה בהגשות

### בקטע "רקע ותיאור הבעיה":

```
דוגמה — הגשה לקרן בנושא נוער בסיכון:

"לפי דוח המועצה לשלום הילד (2025), כ-375,000 ילדים ונוער בישראל
מוגדרים בסיכון. מתוכם, כ-52,000 נמצאים בטיפול חוץ-ביתי (משרד
הרווחה, 2024). שיעור הנשירה מבתי הספר בפריפריה עומד על 8.2%
לעומת 3.1% במרכז (משרד החינוך, מיצ"ב 2024).

מחקר של מכון ברוקדייל (2024) מצא כי תכניות ליווי אישי מפחיתות
נשירה ב-40% ומגדילות שילוב בתעסוקה ב-35% תוך שנתיים."
```

### הכללים:
1. **כל מספר = מקור מפורש** (שם מוסד, שנה)
2. **רק 2024-2026** — לא ישן מ-3 שנים
3. **אם אין נתון מעודכן** → "לפי הנתון האחרון הזמין (2023)..."
4. **אם לא יודע** → "לא מצאתי נתון מעודכן — כדאי לבדוק ב-[מקור]"
5. **לעולם לא ממציא מספרים!**

## עיקרון מפתח: הנתונים תמיד שם — Goldfish לא "שולף", הוא פשוט יודע

**הנתונים נטענים תמיד ל-system prompt** כחלק מ-Promise.all, בדיוק כמו companiesIndex ו-grantsIndex.
לא צריך טריגר מיוחד. לא צריך לזהות מתי המשתמש "צריך" נתונים.

**למה?**
- ~30 נתונים × ~50 תווים = ~1,500 תווים. בורגליים מתוך 180K מותרים
- Goldfish פשוט *יודע* את המספרים כמו מומחה שקרא את כל הדוחות
- משתמש בהם מתי שצריך: בהגשה, בניתוח, בשאלה, או פרואקטיבית

**הזרימה:**
```
ארגון עם DNA: categories=['youth','welfare'], regions=['south']
    ↓
api/chat/route.ts — Promise.all([
  ...6 agents הקיימים...,
  loadResearchContext(orgDNA),  // ← נוסף כאן, סוכן ידע #7
])
    ↓
System prompt מכיל:
"===== נתוני מחקר מאומתים =====
[נוער בסיכון] 375,000 ילדים בסיכון (המועצה לשלום הילד, 2025)
[נשירה] 8.2% נשירה בפריפריה (משרד החינוך, 2024)
[בריאות נפשית] 40% עלייה בפניות אחרי 7.10 (משרד הבריאות, 2024)
..."
    ↓
Goldfish משתמש בנתונים באופן טבעי בכל שיחה
```

## מימוש טכני

### Option A: Knowledge Base ב-Supabase (מומלץ)

```sql
CREATE TABLE research_data (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  topic TEXT NOT NULL,           -- 'youth_at_risk', 'domestic_violence', etc.
  subtopic TEXT,                 -- 'dropout_rates', 'shelters', etc.
  fact TEXT NOT NULL,            -- "375,000 ילדים בסיכון"
  value NUMERIC,                -- 375000 (for sorting/filtering)
  unit TEXT,                     -- 'people', 'percent', 'NIS'
  source_name TEXT NOT NULL,     -- "המועצה לשלום הילד"
  source_type TEXT NOT NULL,     -- 'government', 'academic', 'ngo_data'
  source_url TEXT,               -- link to original
  year INT NOT NULL,             -- 2025
  verified BOOLEAN DEFAULT false, -- manually verified?
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_research_topic ON research_data(topic, year DESC);
```

**הטענה ל-system prompt:**
```typescript
async function loadResearchContext(orgDNA: OrgDNA): Promise<string> {
  // Match research data to org's topics
  const topics = mapCategoriesToResearchTopics(orgDNA.categories);

  const { data } = await supabase
    .from('research_data')
    .select('*')
    .in('topic', topics)
    .gte('year', 2024)
    .eq('verified', true)
    .order('year', { ascending: false });

  if (!data?.length) return '';

  const lines = data.map(d =>
    `[${d.topic}] ${d.fact} (${d.source_name}, ${d.year})`
  );

  return `\n\n===== נתוני מחקר מאומתים =====\n${lines.join('\n')}`;
}
```

### Option B: System Prompt Static (מהיר לביצוע)

הזרקת נתונים קבועים ל-system prompt, מעודכנים ידנית כל רבעון.

### Option C: RAG על מאמרים (עתידי)

העלאת מאמרים אקדמיים ל-Smart Reader → chunks → semantic search per topic.

## עדכון נתונים — מחזור רבעוני

```
כל רבעון (ינואר, אפריל, יולי, אוקטובר):
1. בדוק דוחות שנתיים חדשים (ביטוח לאומי, הלמ"ס, רווחה)
2. חפש מאמרים אקדמיים חדשים (Google Scholar, ResearchGate)
3. עדכן research_data table
4. סמן verified=true אחרי בדיקה
5. מחק/עדכן נתונים ישנים (מעל 3 שנים)
```
